# 策略监控与停止条件

## 基准

每个可使用版本必须冻结：代码 SHA、参数集 ID、数据版本、chronological holdout 指标、成本情景、交易数、平均/中位 R、最大回撤、Profit Factor、候选取消原因分布。样本不足的基准只能标为 `insufficient_evidence`，不能建立强统计承诺。

## 状态

- `normal_variation`：数据门控通过；累计实现结果和取消分布仍在预登记基准范围；没有账户硬风控触发。单独几笔亏损不自动改变参数。
- `performance_degradation`：在至少 30 笔新关闭模拟交易或预登记观察窗后，滚动期望值为负且低于基准置信范围，或 Profit Factor 持续低于 1，或取消/数据失败率显著超过基准。此状态只触发审查，不自动重优化。
- `paused`：账户回撤达到 10%、日亏损 2%、当日止损次数达到 2、组合状态无法核对、行情陈旧/不可用、审计恢复失败，或数据完整性 hash 不匹配。暂停时不创建新候选；已有模拟仓位仍按既定保护规则管理。
- `research_required`：退化在一个完整预登记观察窗后仍存在；策略表现只在单点参数成立；成本翻倍后结果崩溃；数据源或市场结构发生重大变更；或实现/回测假设发生破坏兼容的改变。

## 监控字段

- 数据：fresh/stale/missing/invalid、缺口、重复、规则缓存年龄、circuit 状态。
- 策略：候选数、执行率、取消原因、no_trade 原因、趋势环境、信号分数分布。
- 风险：initial/remaining/aggregate open risk、realized daily loss、current/maximum drawdown、可用/保留现金。
- 表现：滚动交易数、期望值、Profit Factor、胜率（仅辅助）、平均/中位 R、费用、滑点和暴露率。
- 系统：命令失败、重复 command_id、版本冲突、恢复次数、事件/物化状态核对结果。

## 处理原则

1. 先区分数据故障、代码故障、执行假设变化和策略失效。
2. 安全锁或状态不一致立即失败关闭；性能退化只暂停新候选，不删除历史或重写事件。
3. 不因连续两三笔亏损改参数，不根据近期胜率加仓。
4. 任何规则变更创建新策略版本和参数集 ID，旧结果保留；恢复使用前重新通过数据质量、无前视、成本、账户状态和 holdout 测试。
5. 项目无真实下单，因此“暂停”仅指停止输出可执行候选/模拟开仓，不代表对交易所账户采取动作。
